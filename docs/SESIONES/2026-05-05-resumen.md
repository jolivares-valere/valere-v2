## Decisiones arquitectónicas tomadas

1. **Handoff cambia responsabilidad, no visibilidad.** El creador conserva acceso al caso + capacidad de comentar aunque ya no sea responsable. Modelo profesional CRM. Validado con ChatGPT.

2. **Comentarios = actividad tipo `nota`.** Reciclar el tipo existente en lugar de crear `comentario_interno` nuevo. Título "Comentario interno" lo distingue visualmente en timeline. Sin breaking changes en BD.

3. **Footer de cambio de etapa oculto en modo Solo seguimiento.** Carolina A no debe poder pisar el flujo de Carolina M (analista). Se decidió ocultar botones, no mostrar deshabilitados — más limpio.

4. **Helper de vencimiento con campos `estado/diasRestantes`** (no `color/dias`) en `utils/vencimiento.ts`. Más explícito. Tests con `vi.useFakeTimers`.

5. **Caso borde 90 días = amarillo** (NO verde). Comercialmente 90 días es el inicio del seguimiento. Validado.

6. **Sin badge en cards cuando no hay fecha.** Ruido cero para casos sin fecha registrada. Decisión Juan + ChatGPT.

7. **Edge Function ask-crm-docs sin "Fuentes:".** El LLM no debe citar archivos ni rutas. El array `sources` viaja en JSON pero no se renderiza. Tono ayuda integrada, no informe técnico.

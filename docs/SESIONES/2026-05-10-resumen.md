# Sesión 2026-05-10 — FV Sync: diagnósticos completos

## Contexto de entrada

La sesión venía del contexto donde se había descubierto que **CloudWAF de FusionSolar
vincula las cookies de sesión (`HWWAFSESID/HWWAFSESTIME`) al fingerprint TLS del browser**.
httpx tiene fingerprint diferente al de Chrome → WAF rechaza siempre aunque las cookies
sean válidas. Solución adoptada: usar Playwright headless en CI con session state pre-cargado
(`StorageStateClient`), sin hacer login en CI.

El run anterior (#94) falló. Los logs no eran suficientemente descriptivos para saber qué falló.

## Qué se hizo

### 1. `fusionsolar_client.py` — StorageStateClient robusto

- Añadidas excepciones tipadas `FusionSolarAuthError` y `FusionSolarResponseError`
- `_log_storage_state_info()`: loguea nombres de cookies, dominios, claves localStorage
  sin exponer valores. Verifica presencia de cookies críticas: roarand, JSESSIONID, HWWAF, dp-session
- `check_session()`: llamada mínima al portal, devuelve `OK | AUTH_REDIRECT | NON_JSON_RESPONSE | HTTP_ERROR | EXCEPTION`
- `login()`: navega a `base_url + "/"` (no nologin.html), llama check_session() tras navegar,
  lanza `FusionSolarAuthError` si detecta redirect a login, hace screenshot en fallo
- `_fetch()`: JS interno devuelve dict estructurado `{__error: "AUTH_REDIRECT/NON_JSON/HTTP_ERROR", ...}`,
  Python convierte a excepciones tipadas. Ya no puede confundirse HTML con JSON
- `_safe_fetch()`: re-lanza FusionSolarAuthError y FusionSolarResponseError (antes las tragaba)

### 2. `extract_cookies.py`

- Cambiado de `context.cookies()` → `context.storage_state()` para capturar también
  localStorage (tokens adicionales que el SPA de FusionSolar puede usar)

### 3. `sync_job.py`

- Importa `FusionSolarAuthError, FusionSolarResponseError`
- `try/except` del login: catch específico de `FusionSolarAuthError` → log `AUTH_REDIRECT`
  + `ACCIÓN REQUERIDA: ejecuta extract_cookies.py` (antes: error genérico sin indicación de qué hacer)
- Lo mismo en el sync loop si la sesión expira a mitad

### 4. `test_cookie_auth.py`

- Reescrito completamente para usar `StorageStateClient` (el mismo cliente que el CI)
- Llama a `check_session()` explícitamente antes de `get_station_list()`
- Diagnósticos claros: OK / AUTH_REDIRECT / NON_JSON_RESPONSE / SUPABASE_ERROR / UNEXPECTED_ERROR

### 5. `.github/workflows/fv-sync.yml`

- Eliminado step `Instalar Playwright + Chromium` duplicado (estaba dos veces)
- Añadido step "Diagnóstico de entorno" que imprime: fecha/hora UTC, versión Python,
  runner OS, presencia de cada secret (SI/NO sin mostrar valores)

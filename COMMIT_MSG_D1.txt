== Frontend ==

A) Helper migrado a fichero propio + tests
- src/features/captacion/utils/vencimiento.ts (NUEVO):
  * type EstadoVencimiento ('sin_fecha' | 'vencido' | 'rojo' | 'naranja' | 'amarillo' | 'verde').
  * type Semaforo { estado, diasRestantes, label }.
    Cambio de naming respecto al helper viejo (color → estado, dias → diasRestantes)
    para que sea más explícito. Reglas idénticas:
      <0 vencido / <=30 rojo / <=60 naranja / <=90 amarillo / >90 verde.
    90 días entra en amarillo (caso borde acordado).
  * function siguienteAccionLead(etapa, fechaVencimiento):
    Mapeo base por etapa_operativa, con OVERLAY de urgencia si la fecha
    está cerca:
      vencido → "Urgente: contrato vencido — llama ya"
      rojo    → "Urgente: vence en X días — llama ya"
      naranja → "Prioridad alta: vence en X días"
      amarillo→ "Contactar ya: vence en X días"
      verde   → texto base de etapa (sin overlay)
      sin_fecha → texto base de etapa
    Etapa "cerrado" nunca aplica overlay.
  * const ESTADO_CLASSES: mapeo central de clases Tailwind por estado,
    para que ningún consumidor las redefina y diverjan.

- src/features/captacion/utils/vencimiento.test.ts (NUEVO):
  * 7 tests calcularSemaforoVencimiento (null, vencido, 15d, 50d, 80d, 90d=amarillo, 200d).
  * 8 tests siguienteAccionLead (etapas + overlay urgencia + cerrado sin overlay).
  * Usa vi.useFakeTimers() con fecha fija 2026-05-05.

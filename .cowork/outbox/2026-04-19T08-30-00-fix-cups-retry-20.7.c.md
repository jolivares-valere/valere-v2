# Fix cups.contrato_id nullable + Retry FASE 20.7.c

## Contexto

Tu intento de migrar `supply_points` → `cups` falló porque
`cups.contrato_id` era `NOT NULL` y `supply_points` no tiene equivalente.

Ya he creado la migración: `supabase/migrations/20260419_fix_cups_contrato_nullable.sql`

## Instrucciones

1. Haz `git pull origin claude/valere-crm-architecture-2vvEV` para traer la migración.
2. Aplica la migración:
   ```sql
   ALTER TABLE public.cups ALTER COLUMN contrato_id DROP NOT NULL;
   ```
3. Reintenta la migración de supply_points → cups:
   ```sql
   INSERT INTO public.cups (id, empresa_id, codigo_cups, direccion_suministro,
     distribuidor, estado, created_at)
   SELECT sp.id,
          sp.client_id,
          sp.cups,
          sp.direccion,
          sp.distribuidora,
          'activo',
          sp.created_at
   FROM public.supply_points sp
   WHERE NOT EXISTS (SELECT 1 FROM public.cups c WHERE c.codigo_cups = sp.cups);
   ```
4. Verifica:
   ```sql
   SELECT COUNT(*) FROM cups WHERE codigo_cups IN (SELECT cups FROM supply_points);
   ```
5. Si todo OK, podemos considerar DROP de `supply_points` y `clients` en una
   futura iteración (no hagas DROP ahora).

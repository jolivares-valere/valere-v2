# FASE 20.8 — Chat Gemini → Edge Function (quitar API key del cliente)

## Contexto

El componente `src/features/chat-ia/ChatIAPanel.tsx` usa
`@google/genai` directamente en el navegador con la API key expuesta vía
`import.meta.env.VITE_GEMINI_API_KEY`. Esto es un agujero de seguridad:
cualquiera que abra el bundle puede extraerla.

Solución: mover la llamada a una Edge Function de Supabase que:
- Recibe `{ messages: [{ role, content }] }` en el body.
- Llama a Gemini con la API key leída desde `Deno.env.get('GEMINI_API_KEY')`
  (secret de Supabase, NO variable de entorno del cliente).
- Devuelve la respuesta tal cual.

## Tareas

### 1. Crear Edge Function

Fichero nuevo: `supabase/functions/chat-consultor/index.ts`

```typescript
import { serve } from 'https://deno.land/std@0.208.0/http/server.ts'
import { GoogleGenAI } from 'npm:@google/genai@1.0.0'

const GEMINI_API_KEY = Deno.env.get('GEMINI_API_KEY')
if (!GEMINI_API_KEY) {
  console.error('Missing GEMINI_API_KEY secret')
}

const corsHeaders = {
  'Access-Control-Allow-Origin': '*',
  'Access-Control-Allow-Headers': 'authorization, x-client-info, apikey, content-type',
  'Access-Control-Allow-Methods': 'POST, OPTIONS',
}

serve(async (req) => {
  if (req.method === 'OPTIONS') return new Response('ok', { headers: corsHeaders })
  if (req.method !== 'POST') {
    return new Response(JSON.stringify({ error: 'Method not allowed' }), {
      status: 405, headers: { ...corsHeaders, 'Content-Type': 'application/json' }
    })
  }

  try {
    const { messages, systemPrompt } = await req.json()

    const ai = new GoogleGenAI({ apiKey: GEMINI_API_KEY! })
    const model = ai.getGenerativeModel?.({ model: 'gemini-2.0-flash-exp' })
      ?? ai.models.generateContent

    // Adaptar mensajes al formato de Gemini
    const contents = messages.map((m: any) => ({
      role: m.role === 'assistant' ? 'model' : 'user',
      parts: [{ text: m.content }],
    }))

    const response = await ai.models.generateContent({
      model: 'gemini-2.0-flash-exp',
      contents,
      config: systemPrompt ? { systemInstruction: systemPrompt } : undefined,
    })

    const text = response.text ?? response.candidates?.[0]?.content?.parts?.[0]?.text ?? ''

    return new Response(JSON.stringify({ text }), {
      headers: { ...corsHeaders, 'Content-Type': 'application/json' }
    })
  } catch (err) {
    console.error('chat-consultor error:', err)
    return new Response(JSON.stringify({ error: (err as Error).message }), {
      status: 500, headers: { ...corsHeaders, 'Content-Type': 'application/json' }
    })
  }
})
```

### 2. Desplegar

```bash
supabase secrets set GEMINI_API_KEY=<valor>
supabase functions deploy chat-consultor
```

Confirmar con:
```bash
curl -i -X POST https://<PROJECT_REF>.supabase.co/functions/v1/chat-consultor \
  -H "Authorization: Bearer <ANON_KEY>" \
  -H "Content-Type: application/json" \
  -d '{"messages":[{"role":"user","content":"Hola"}]}'
```
